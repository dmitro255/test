
Этот проект содержит кастомные реализации основных методов работы с массивами в JavaScript. Код написан на чистом JS с использованием циклов `for`, что позволяет понять внутреннюю логику работы стандартных методов `Array.prototype`.

## Содержание функций

В библиотеке реализованы следующие хелперы:

1.  **printArray / printArray1** — функции для форматированного вывода элементов массива в консоль.
2.  **forEach(array, callback)** — выполняет переданную функцию для каждого элемента.
3.  **map(array, callback)** — создает новый массив, трансформируя каждый элемент.
4.  **filter(array, callback)** — возвращает новый массив только с теми элементами, которые прошли проверку.
5.  **find(array, callback)** — находит и возвращает первый элемент, удовлетворяющий условию.
6.  **some(array, callback)** — возвращает `true`, если хотя бы один элемент подходит под условие.
7.  **every(array, callback)** — возвращает `true`, только если ВСЕ элементы подходят под условие.
8.  **reduce(array, callback, initialValue)** — сжимает массив до одного значения (например, суммы).

## Примеры использования

```javascript
const numbers = [1, 2, 3, 4, 5];

// Трансформация (map)
const squared = map(numbers, (el) => el * el); // [1, 4, 9, 16, 25]

// Фильтрация (filter)
const evens = filter(numbers, (el) => el % 2 === 0); // [2, 4]

// Агрегация (reduce)
const sum = reduce(numbers, (acc, el) => acc + el, 0); // 15

🧠 Теоретическая справка
1. Преимущества использования колбэков
Абстракция: Мы отделяем механизм обхода массива (цикл) от конкретной логики обработки данных.
Декларативность: Код становится понятнее. Глядя на слово filter, сразу ясно, что происходит фильтрация, в то время как обычный цикл for требует вчитывания в логику.
Переиспользование: Одна функция (например, map) может выполнять тысячи разных задач, просто принимая разные колбэки.
2. Проблемы и их решение
Callback Hell: Сильная вложенность функций делает код нечитаемым.
Решение: Использовать именованные функции или современные Promises / async-await.
Потеря контекста (this): Обычные функции могут терять доступ к объекту, в котором они вызваны.
Решение: Использовать стрелочные функции () => {}, так как они не имеют своего this.
Отладка: Анонимные функции сложнее найти в стек-трейсе ошибок.
Решение: Давать колбэкам имена при необходимости.
3. Принцип реализации без встроенных методов
Все функции построены на базе цикла for. Основные шаги:
Создание результирующей переменной (массив или значение).
Проход по исходному массиву по индексам.
Вызов callback(element, index, array) на каждой итерации.
Логическое ветвление:
В filter добавляем в результат, если колбэк вернул true.
В find возвращаем элемент сразу при первом true.
В reduce перезаписываем аккумулятор результатом выполнения колбэка.
